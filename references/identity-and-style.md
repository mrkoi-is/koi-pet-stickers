# 身份锁定与表达边界

分析宠物照片或编写宫格提示词前，先读本文件。本文件只锁定“是谁”和“不能变成什么”；情绪如何演、是否用动作、显示什么文字，由所选风格和运行计划决定。

## 只记录可见身份事实

从照片中提取：

- 物种；只有视觉证据充分时才写品种；
- 脸型、吻部 / 喙、耳朵 / 冠羽 / 角、眼睛和鼻子；
- 主体毛色、羽色或皮肤色及其位置；
- 显著斑块、条纹、白袜、额线、面罩或不对称特征；
- 照片里可见的身体比例、尾巴和稳定配饰；
- 看不见或不确定的区域。

从这些事实中选出 3–6 个身份锚点，至少覆盖物种与头部轮廓、头部附属结构、眼睛与鼻子 / 喙 / 吻部、主色和高显著花纹或色块。品种名不能代替这些可见锚点；没有证据的类别写入不确定项。

再把最关键的 3–4 个高显著结构压缩成一行 `IDENTITY_CHECKSUM`：物种原生头部结构 + 耳型结构 / 耳根连接或不对称 + 吻部 / 喙 + 主色 / 高显著色块。它必须能在结构清晰可见时逐项核验，不能写“可爱、蓬松、某品种感”等无法核验的套话。合理裁切、透视或遮挡造成的不可见项记为 `N/A-occluded`，不能当成身份丢失。

## 把身份翻译进风格

宠物照片控制“是谁”，风格预设控制“怎么画”。为每个身份锚点写出风格兼容的等价画法：用稳定外轮廓家族保存脸型，用耳朵数量、耳根连接和耳廓基本类型保存耳型结构，用大色块地图保存花纹，用眼距、眼周色块或吻部位置保存神态。不得为了套用通用萌宠脸而牺牲身份。

风格与真实身份冲突时，先保留物种结构、外轮廓家族、先天耳型结构、花纹地图、吻部 / 喙、主色和不对称标记，再用所选风格允许的简化、扁平、稚拙或怪萌夸张表达。禁止把所有物种都改成同一种狗鼻子、猫脸、尖耳或圆脸模板。

## 分离耳型结构与动态耳姿

耳朵身份锁只锁定先天结构，不把参考照片那一刻的耳位冻结成模板：

- **耳型结构：** 耳朵数量、耳根连接位置、耳廓基本类型与静息耳廓长度级别、稳定不对称和高显著耳部花纹；这些决定“还是不是同一只宠物”；
- **动态耳姿：** 耳廓向前关注、向侧后方展开的飞机耳、向后贴近头部的背耳、放松下沉、单耳转向，以及由歪头、俯仰和透视造成的缩短或遮挡；这些是情绪与动作语言，可以变化。

合理耳姿不是身份漂移。飞机耳保留耳根和耳廓结构，只改变侧向 / 外向展开；背耳保留同一结构，只把耳廓向后旋转或贴近头部。强夸张风格可以压扁、拉开、错位或透视缩短耳姿以增强艺术价值；投影长度变化不等于静息耳廓长度级别改变。不能改变耳朵数量、替换耳根连接、把原有耳廓换成另一种先天耳型，或在清晰可见时系统性丢失高显著耳部花纹。

“飞机耳”和“背耳”只作为便于交流的外观别名，不是跨物种通用的科学分类。运行时和 QA 都按几何事实描述：侧向 / 外向展开或压低，与向后 / 向头骨贴近分开记录；不同物种和先天耳型允许的运动幅度不同，不能拿猫的耳姿模板硬套给犬、兔、鸟或其他宠物。

单一耳位不能独立判定情绪；必须结合眼口、头部与躯干姿态、尾巴 / 翅膀和语境综合判断。只有明确的耳型结构替换才进入身份 gate；看起来介于夸张耳姿与结构变化之间、但仍能认出同一只宠物时记为 warning，不立即整图重生。被合理裁切、透视缩短或遮挡的锚点记 `N/A-occluded`；只有可见结构与照片事实矛盾，或同一锚点在多个清晰非遮挡视图中持续错误，才算身份硬失败。

参考照片只锁定可见身份锚点，不限制构图范围。最终可使用头部、肩部近景、半身或全身；未见区域可以按当前物种正常结构与正常肢体数量保守补全，并延续已知主色与大色块。不要把推断区域写成身份事实，也不要虚构高显著隐藏花纹、特殊配饰或新的不对称标记。

## 可选风格参考图

用户主动提供风格参考图时，把输入分成两组：

- **身份参考：** 当前宠物照片，唯一控制物种、个体轮廓、耳型、五官位置、毛色、花纹和可见身份锚点；
- **风格参考：** 只提取构图档、线条粗细与节奏、色彩数量与覆盖率、留白比例、材质、夸张幅度、文字笔触和轮廓语言。

风格参考图不得贡献角色身份、物种、品种、花纹、配饰、具体文字、标点、道具、姿势、格子顺序、布局、场景或背景，也不得复刻具体角色或受保护 IP。没有风格参考图时使用稳定预设，不向小白主动索要。

强夸张风格可以改变眼睛、鼻子、吻部、头部和身体的相对比例，也可以使用合理的飞机耳、背耳或透视耳姿；但必须保留物种结构、耳根连接、先天耳型、主色地图、高显著花纹和不对称标记。犬类只夸张犬类吻部，猫科只夸张猫科脸型与胡须结构，鸟类只夸张喙与冠羽等物种原生锚点，禁止跨物种套用同一张怪萌脸。

## 锁定风格、构图与文字

从 [style-presets.md](style-presets.md) 选择一个 `style_id`；用户未指定时使用 `q-cute-handdrawn`。同一套表情保持身份、物种结构、绘画语言和色彩映射一致，但裁切、头身比、尺度和形变可按所选构图档受控变化。

构图只服从预设声明的档位：

- `face-first`：优先用头像、肩部近景、脸部变形与局部裁切表达；
- `balanced`：在头像、半身和紧凑全身之间形成节奏；
- `action-first`：优先用动态半身 / 全身、方向与动作线表达。

文字由 `TEXT_POLICY`、`TEXT_LANGUAGE` 和 `RENDERED_TEXT_PLAN` 控制。用户可以指定全中文、全无字、混合、其他语言或自定义文案；未指定时使用所选风格的默认文字策略。16 个固定中文 semantic 只用于内部顺序、稳定 ID、文件名和 manifest，不是必须画进图里的中文文字。

`RENDERED_TEXT_PLAN` 必须为每个稳定 ID 写一个精确字符串或 `NONE`：

- `NONE` 格禁止汉字、字母、数字、乱码和伪文字；
- 非空格只能出现计划中的精确字符串，不能多字、少字或换成近义词；
- 即使有字，情绪也要能靠角色、裁切、姿态或装饰独立成立；文字不能补救空洞表演。

## 情绪语义优先，动作不标准化

`CELL_SEMANTICS` 只包含 16 个稳定 ID、内部中文语义标签与核心情绪方向；它不是动作清单，也不是可见文字清单。

每格必须把对应情绪语义传递正确，但不要求使用统一动作。模型可以按风格选择脸部形变、眼口关系、歪头、俯仰、肩部变化、半身或全身姿势、道具、裁切和贴近角色的情绪装饰；只要遮住文字后语义仍可理解即可。情绪装饰无需在 `CELL_SEMANTICS` 逐项声明，不能因为某格没有指定泪滴、爱心、问号或动作线就判失败。

所有动作遵守当前物种结构，禁止人手、五指、握拳、比赞、OK 手势、敬礼、合十或补造肢体。允许补全符合当前物种正常结构与正常肢体数量的简化身体；这不算补造肢体。不得把爪、翅、鳍或蹄改造成手，也不得为了聊天手势增加额外肢体。

16 格不能只是同一正脸更换眉毛、文字或道具。`face-first` 允许大量近景，但仍要通过正面 / 侧面 / 三分之四角度、俯仰、歪头、局部裁切、轮廓变形和眼口关系形成差异；不得让同一正脸模板重复超过四格。`balanced` 与 `action-first` 则按各自构图档改变头像、半身、全身和方向节奏。

## 运行时生成锁

提示词必须明确：

- 当前宠物的身份事实、不确定项、3–6 个身份锚点和一行 `IDENTITY_CHECKSUM`；
- 当前宠物的 `EAR_MORPHOLOGY_LOCK`：先天耳型结构不可替换，动态耳姿可按物种合理性和表达需要变化；
- 选定的 `STYLE_LOCK`、`COMPOSITION_PROFILE`、`COMPOSITION_LOCK`、`ANTI_STYLE_LOCK` 与提取兼容要求；
- 固定 `4×4` row-major 映射和 `CELL_SEMANTICS`；
- `TEXT_POLICY`、`TEXT_LANGUAGE`、`CAPTION_LOCK` 和逐格 `RENDERED_TEXT_PLAN`；
- 情绪语义必须正确，但动作、裁切、道具和装饰不标准化，并优先服从所选风格；
- 每格为完整、居中、适合裁切的同格贴纸簇；允许计划文字或少量装饰与角色断开，但必须明确归属同格；
- 整张画布统一纯白色（#FFFFFF），禁止 logo、水印、场景、地面、对话气泡、松散背景装饰、未计划文字和游离道具；
- 可选风格参考图只控制抽象画法，禁止复制其中的角色、文字、道具、布局或背景。

## 失败与修复

默认阶段把身份漂移、物种结构 / 肢体、情绪、计划文字、风格、构图档和重复模板记录为非阻断视觉观察；只把透明切图缺失、裁切、错格、邻格残留、白底方块或透明化误删记为交付失败。

透明化失败先重新提取；只有源图无法完整分离成 16 张时，才允许把上一张作为布局参考整图修复一次。不得做单格补画、拼接或本地改字；第二张仍无法完整切图就停止。缩放或打包错误只重跑对应下游步骤。用户明确要求完整标准包时，再启用严格身份、文字、语义和风格 QA。
