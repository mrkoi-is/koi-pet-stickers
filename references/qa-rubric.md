# QA 规则

展示 REDSkill 大宫格、透明单格或宣称最终交付前，先读本文件。默认阶段采用“切图成功优先”：先验证透明提取，身份、情绪、风格、动作和文字只做观察记录。只有用户明确要求完整标准包时，才把后续严格视觉 QA 作为交付 gate。

## 判定层级

- **默认切图硬失败：** `extract_grid.py` 返回非零或 `ok: false`；没有正好 16 张非空 RGBA PNG；透明预览确认任一角色、文字、耳朵、爪子、尾巴、白边、道具或装饰被裁掉、错格、混入邻格、残留白底方块或被透明化误删。只有实际透明结果失败才触发整图修复。
- **默认非阻断 warning：** 所有脚本 warnings，以及身份、情绪、风格、动作、耳姿、构图和文字效果差异。逐格记录观察，但只要 16 张透明结果完整、顺序正确且互不污染就继续交付。
- **完整标准包源图硬失败：** 用户明确要求完整标准包时，再把身份漂移、物种结构 / 肢体错误、情绪语义错误、计划文字错误、风格反指纹、logo、水印和其他严格视觉问题升级为阻断项。
- 边缘抗锯齿、PNG 压缩色阶或极小的纯色眼睛高光不算渐变或写实渲染。`flat-emoji` 中一块边界清楚的次级纯色暗面可计入 4–6 个色块；连续软过渡、喷枪体积和可见毛丝仍不允许。

安全问题始终不能交付。其余视觉差异在默认阶段只记录；只有它们导致实际切图缺失、错格、邻格污染或透明化误删时才阻断。为构图补全符合当前物种正常结构与正常肢体数量的简化身体，不属于补造肢体。

## QA 1：REDSkill 大图

以下项目用于视觉观察和完整标准包审核；默认切图阶段不因其中任一项单独阻断：

- 16 格看起来是同一只宠物、同一种风格；
- 物种、脸部轮廓、耳朵、眼睛、吻部 / 喙、鼻子、主色和高显著花纹能从原图里认出来；
- 预先记录的 3–6 个身份锚点在结构清晰可见时保持；耳朵数量、耳根连接、先天耳型、静息耳廓长度级别、稳定不对称和高显著耳部花纹不被替换；合理裁切、透视或遮挡造成的不可见项记为 `N/A-occluded`，不算身份丢失；
- 动态耳姿可按物种能力、情绪和风格改变。向侧后方展开 / 压低的飞机耳、向后旋转并贴近头部的背耳、前倾、放松下沉、单耳转向、透视缩短或部分遮挡都可通过；合理耳姿不是身份漂移；
- 单一耳位不能独立判定情绪；必须结合眼口、头部和躯干姿态、尾巴 / 翅膀、装饰与上下文综合判断。不能因“委屈没有背耳”或“参考照是立耳”单独判失败；
- 所有动作遵守当前物种结构，禁止人手、五指、握拳、比赞、OK 手势、敬礼、合十或补造肢体；
- 固定 row-major 映射正确；稳定 ID 与内部中文语义标签只负责顺序、文件名和交付记录，不自动成为画面文字；
- 每格的核心情绪语义清楚、与 `CELL_SEMANTICS` 一致，并且即使遮住文字仍能从脸部、轮廓、姿态、裁切或装饰中理解；
- 不要求某个情绪对应固定动作。动作、视角、裁切、道具和装饰按所选风格自由选择；只要语义正确、物种结构正确、画面有表达力即可；
- 动作、姿势和装饰的具体形式不作为硬失败；情绪语义缺失、相反或只能靠错位文字解释仍是硬失败；
- 不按情绪装饰是否与 `CELL_SEMANTICS` 一致判定失败；问号、感叹号、爱心、星光、怒气、睡眠符号、动作线等可按语义自由出现，只检查其贴近角色、留在安全区内、不遮挡计划文字且不跨格；
- `RENDERED_TEXT_PLAN` 为 `NONE` 的格不得出现汉字、字母、数字、乱码或伪文字，包括 `Zzz`、`ZZZ`、`Hi`、`OK`、`SOS`、首字母和字母型拟声；问号、感叹号、爱心、星光、箭头、泪滴、汗滴、腮红和动作线属于非词汇情绪符号，可按语义自由使用；计划非空时只能出现该精确字符串，语言由用户指定或所选风格默认策略决定；
- 中文文字仅在计划字符串本身为中文时核对中文；其他语言按对应精确字符串核对；风格默认无字时不因缺少文字失败；
- 每格是一个紧凑、适合裁切的同格贴纸簇。角色、白边、道具和装饰通常相连；计划文字或少量语义装饰可以与角色断开，但必须明显属于该格；
- 各格形成可独立提取的紧凑表达簇，行列之间有可辨认的纯白走廊；不以固定 20% 四边留白或绝对居中作为默认通过条件；
- 整张源图背景统一纯白色（#FFFFFF），没有米白、灰白、暖纸色、纹理或光影变化；
- 构图服从所选预设的 `face-first`、`balanced` 或 `action-first` 档，不用通用全身动作模板覆盖风格；
- 遮住文字和道具后，各格仍有符合构图档的脸部形变、视角、裁切、姿态或轮廓差异；同一正脸模板不得重复超过四格；
- 均分裁切后不会出现邻格残留；没有 logo、水印、场景、地面阴影、对话气泡或松散背景装饰。

身份和风格必须分开审核：先只看照片与角色，确认“是谁”；再忽略宠物身份，确认构图档、线条、色彩、比例、质感、留白和文字层级是否符合 `style_id`。风格通过不能抵消身份漂移，像本人也不能抵消风格硬失败。

### 身份锚点矩阵

不能凭整体相似直接通过。审核者必须建立 16 行 × 3–6 列的身份锚点矩阵：每行对应一个格子，每列对应一个预先记录的锚点，并逐项标记通过 / 失败 / `N/A-occluded`。合理裁切、头部转向、透视缩短或遮挡造成的不可判定只能记 `N/A-occluded`；只有可见结构与照片事实矛盾，或同一锚点在多个清晰非遮挡视图中持续错误，才是身份硬失败。耳部拆成两列审核：

- **耳型结构列：** 耳朵数量、耳根连接、先天耳型、静息耳廓长度级别、稳定不对称和高显著耳部花纹；投影长度变化不等于先天长度变化。明确换成另一种结构、凭空增减耳朵，或在清晰可见时系统性丢失身份花纹才是硬失败；
- **动态耳姿列：** 前倾、侧向 / 外向展开的飞机耳、向后贴近头部的背耳、放松下沉、单耳转向、透视缩短和遮挡；只要能从同一耳根与耳廓类型追溯回来就通过，艺术夸张较强但仍可辨认时记 warning；
- **艺术让步：** `bold-ink-caricature` 可进一步压扁、错位、拉开或部分隐藏耳廓以换取怪萌轮廓和情绪强度。只有艺术处理已经造成耳型结构替换、角色变成另一只宠物或另一物种时才硬失败。

另把 `IDENTITY_CHECKSUM` 中的物种原生头部结构、耳型结构 / 耳根连接或高显著不对称、吻部 / 喙和主色地图作为高优先级列。镜像稳定不对称、替换吻部 / 喙、丢失高显著色块或退化成通用宠物模板仍是身份硬失败；不得用“其他格子很像”抵消。

### 构图档 QA

- **`face-first`：** 主要靠头像、肩部以上近景、脸部变形、局部裁切和眼口关系传达；少量半身 / 全身只能用于必要语义。若大多数格退化成同一比例的完整全身动作贴纸，判定构图档失败。
- **`balanced`：** 头像、半身和紧凑全身有明确节奏，既不能 16 格全是同一正脸，也不能 16 格全是同一尺度全身。语义相近的格子也要在裁切、视角或轮廓上拉开。
- **`action-first`：** 主要靠动态半身 / 全身、方向、速度线和强轮廓传达；脸部仍须可读，文字不能代替动作。若大多数格只是静态头像换字，判定构图档失败。

### 六种风格指纹与反风格

先在整张缩略图核对正向指纹，再抽查每格硬约束。不能只凭“可爱、手绘、水彩、漫画”等大类词通过。

#### `q-cute-handdrawn`（`balanced`）

- **正向指纹：** 圆润 Q 萌比例、清楚中等粗描边、暖而克制的平涂、少量手绘抖动与统一白边；头像、半身和紧凑全身有平衡节奏；默认混合文字策略只在需要聊天语气的格子使用短字。
- **反风格：** 写实毛发侵入、连续柔和明暗、照片式体积光、同一全身模板重复、角色被大标题压缩成小图，或每格只换眉嘴。4 格以上主导即硬失败。

#### `flat-emoji`（`face-first`）

- **正向指纹：** 面部与肩部近景占主导；每个角色由 4–6 个边界清楚的纯色色块构成；几何轮廓、统一硬边、零纹理、零渐变、可选硬边阴影；默认 16 格无字，靠表情与符号成立。
- **反风格：** Q 萌毛发贴纸、绘画笔触、彩铅 / 水彩纹理、软阴影、三维体积、10 格以上完整全身，或写实毛色渐变。4 格以上主导即硬失败。

#### `bold-comic`（`action-first`）

- **正向指纹：** 动态半身 / 全身、强方向感、粗黑轮廓、高对比红黄 / 黑黄配色；有字时大字是第一视觉层级，但遮住文字后动作和情绪仍成立。
- **反风格：** 小而柔和的手写字、静态对称站姿、轻柔水彩、细线萌宠肖像、弱对比配色，或文字不再形成漫画冲击力。4 格以上主导即硬失败。

#### `crayon-journal`（`balanced`）

- **正向指纹：** 蜡笔颗粒只在角色与道具色块内部；线条略歪、色块有手工擦涂边缘；画布仍为纯白；头像、半身和紧凑全身交替；默认少量手写短字而非每格大标题。
- **反风格：** 细密彩铅写实毛发、均匀精描肖像、平滑数码线、厚黑漫画线、整画布米色纸纹、胶带拼贴，或 16 格同一全身比例。4 格以上主导即硬失败。

#### `naive-ink-watercolor`（`face-first`）

- **正向指纹：** 稚拙、轻细、断续的墨线；角色内部约 45–60% 稀疏透明水彩，至少 40% 轮廓内白纸；头像与肩部近景占主导，眼口允许轻微错位；默认只有少数松弛手写短字。
- **反风格：** 大面积完整铺色、精致萌系水彩、平滑数码线、亮晶晶大眼、写实毛发、标准矢量图标、12 格以上完整全身，或工整对称脸。4 格以上主导即硬失败。

#### `bold-ink-caricature`（`face-first`）

- **正向指纹：** 怪形优先，动作后置；至少 12 格为头像或肩部以上近景，少于 12 格就是构图硬失败；最多 4 格可用紧凑半身 / 全身。至少 8 格同时具备两个明显不对称维度：眼白大小 / 高低、脸轴倾斜、吻鼻偏移或压缩、鼻头尺度 / 位置、局部裁切；常规对称萌脸超过 4 格就是怪形硬失败。整张缩略图先读成黑白粗墨怪萌漫画，再看到宠物身份色：每格以少而重的宽头粗黑主线建立轮廓，身份色收束为最多 3 组主要水彩色块群；同一色组可分散成多个小岛以保留高显著花纹。使用小黑瞳、无亮晶晶高光和允许错位的不规则眼白；主轮廓线宽、20–40% 水彩和至少 60% 轮廓内白纸是整张与多数格的风格目标，整体必须负空间主导；不要求逐格像素测量，只有相反画法在 4 格以上主导才触发风格硬失败。飞机耳 / 背耳等夸张耳姿可增加艺术价值，不因偏离参考照瞬时耳位扣分。
- **反风格：** 常规可爱全身宠物贴纸、对称标准萌脸、顺滑均匀贴纸描边、完整亮色毛色铺满、精致水彩肖像、亮晶晶大眼、正常比例写实五官、细密短毛排线，或依赖完整全身动作表达。任一核心反风格在 4 格以上主导、头像 / 肩部近景少于 12 格、少于 8 格达到两个明显不对称维度，或常规对称萌脸超过 4 格，均为源图硬失败。

用户提供风格参考图时，再做角色隔离审核：只比较构图档、线重与节奏、色彩数量与覆盖率、留白比例、材质、夸张幅度和文字笔触；不得出现参考图里的角色身份、物种、花纹、配饰、具体文字、符号、道具、姿势、布局或背景。

## QA 1.5：透明单格

默认 REDSkill 路径必须检查透明化结果：

- 输出正好 16 张透明 PNG；
- 每张 PNG 同时有透明像素和不透明像素，外围白底已变透明；
- 每张至少 10% 画布为透明区域，可见内容不触碰画布外边缘，且不存在透明边框包围的不透明矩形底板；
- 白毛、白色贴纸边、浅色道具和计划文字描边没有被透明化吃掉；
- 单张里没有相邻格残留、宫格缝或白底方块；
- 棋盘格预览中，所有非空计划文字完整清楚；`NONE` 格没有残留伪文字；
- `scripts/extract_grid.py` 返回跨格组件、归属接近网格线、比例安全边距不足或同格多个断开组件的 warning 时，必须逐格人工确认，但 warning 本身不阻断。允许同格文字或语义装饰与角色断开，只要透明结果完整且没有邻格残片。
- 正式选择 `source-sheet.png` 前必须先做一次诊断提取。`source_components_cross_grid`、`possible_neighbor_residue`、`component_owner_near_grid_line`、`source_safe_margin_below_ratio`、`source_margin_ratios`、负 `source_margins`、`source_content_near_cell_edge` 和 `multiple_components_in_cell` 都是定位风险的证据，不是自动失败条件；最终以 16 张透明 PNG 和棋盘格预览是否完整、顺序正确、互不污染为准。

透明提取固定使用 `--halo-px 2`。如果误伤白色主体、白边或文字描边，只重新提取并检查源图是否为纯白背景；不要本地补画。只有源图本身失败时才使用一次整图修复机会。

## QA 2：机器验证

标准完整包必须要求 `scripts/validate.py` 返回 `0`，并写出 `ok: true` 的 `qa-report.json`。机器检查覆盖：

- 文件数量和可预测文件名；
- PNG 格式、RGBA 模式、至少 10% 透明区域、非矩形底板、可见内容不触边及标准尺寸；
- 稳定 ID、内部中文语义、`text_policy`、可空 `rendered_text`、源图 hash 格式和输出 hash 一致性；
- 支持的 `style_id` 和显示名；
- 缺失、多余、未约定或重复资产；
- 源图、master、manifest、预览与报告路径碰撞，包括 Unicode / 大小写等价路径和同一文件别名；
- `preview/contact-sheet.png` 是否存在和 hash；
- 元数据是否去除。

机器通过不等于文字准确或身份正确，也不等于情绪语义和风格成立。原生中文文字、其他语言文字或无字策略都必须按 `RENDERED_TEXT_PLAN` 做视觉 QA。

## QA 3：最终视觉复查

标准完整包要在实际尺寸和带棋盘格的 `preview/contact-sheet.png` 里检查 16 张 master：

- 能认出是原始照片里的同一只宠物；
- 情绪语义正确、有表达力；不要求与固定动作逐项一致，也不以动作或情绪符号的种类判定失败；
- 动作遵守当前物种结构与正常肢体数量，没有人手或人类手势；
- 耳朵、爪子、尾巴、白边、计划文字、道具和装饰没有被裁掉；
- 非空 `rendered_text` 与 manifest 完全一致、可读、不乱码；`NONE` 格没有伪文字；
- 透明背景没有误删白毛、白边或文字描边；
- 主体尺度与裁切遵守所选构图档，线条、色彩、留白、材质和文字层级符合所选风格；
- 没有用重复姿势或重复正脸冒充不同语义；
- 没有宫格缝、位置错换、邻格残留或禁用内容。

## 单文件失败记录

每个失败项都要记录：输出文件名与稳定 ID、失败的 QA 门槛、观察到的问题、问题来自源图 / 裁切 / 打包 / 提示词、最小修复动作以及是否需要重新生成。

默认阶段先重跑透明提取；只有确认源图无法完整分离成 16 张时，才允许把上一张作为布局参考整图修复一次。打包、尺寸或元数据失败只重新打包。不得用单格补画、拼接或本地改字修复源图；第二张仍无法完整切图就停止并说明。完整标准包仍须通过本文件的严格视觉复查和机器验证。
